Problem Statement
As developers, we often encounter a wide range of tasks; some involve complex algorithms, while others can be as mundane as mapping, displaying, or updating static data. These repetitive tasks, though essential, can become time-consuming if done manually.

In the development process, these recurring tasks are not uncommon. It's incumbent upon us to find ways to optimize our workload. By doing so, we can allocate more time and focus on the creative and problem-solving aspects of our projects.

Take, for instance, my experience with the Starbucks Drink Recommendation App. I needed to create over 100 different fields for user selection. Entering these manually was an option, but it would have eaten up a significant chunk of my time.



Mapping Use Case
A developer often encounters tasks that, while crucial, can be remarkably tedious. Mapping is a prime example; it involves taking a set of inputs and converting them into a standardized object used elsewhere in your codebase.

Imagine a scenario where a business provides you with a CSV file containing two columns. Column A holds the payload of an object you're ingesting, while Column B indicates the corresponding field in your codebase's schema. The instinctual solution for many developers might be manual mapping - a process that, unfortunately, can consume hours of valuable time.

Example task to preform:

my_class.first_name = their_class.first
my_class.last_name  = their_class.last
my_class.email      = their_class.email_address
...
my_class.phone_cell = their_class.cellphone_number

Instead, consider the power of automation. With just a few lines of code, you can achieve in seconds what might otherwise take hours:

import csv
with open('mapping.csv', "r") as csv_file:
    reader = csv.DictReader(csv_file)
    for row in reader:
        values = row.values()
        print(f'my_class.{values[1]} = their_class.{values[0]}')

This practice of "code writing code" not only provides a significantly faster solution compared to manual mapping, but it also ensures uniformity and reduces the likelihood of errors.



"Code Writing Code" as a Service
Mundane coding tasks may demand mundane solutions, but this use case of "code writing code" also extends into complex situations.

One of my favorite automation tools, though not explicitly for code, is Nils Werner's VSCode extension autoDocstring - Python Docstring Generator. This handy extension acts like a webhook, generating proper docstring templates for your Python code. It ensures that your documentation remains clean and consistent.

Additionally, I find ChatGPT to be an invaluable resource. Manually writing test cases can be a tedious endeavor, but with properly documented code, ChatGPT has proven extremely helpful in automating this task.

On another note, GitHub offers a premium paid feature called GitHub Co-Pilot, which excels in generating code snippets.



Closing Thoughts
Unfortunately, tedious coding tasks are as unavoidable as taxes. Whether you're adept at writing code to generate more code or prefer utilizing available services, there are options to handle these tasks efficiently.

I hope that this blog serves as inspiration for you to look into the many resources that can help you automate your development process!
